## 先了解一下

解压后你会看到这些文件：

```
vertex-proxy（或 vertex-proxy.exe）   ← 这就是主程序，只有这一个
config/
  ├── config.example.json              ← 配置模板
  ├── api_keys.example.txt             ← 密钥模板
  └── models.json                      ← 可用的模型列表（一般不用改）
```

还有一个启动脚本（Windows 是 `启动.bat`，Linux/Android 是 `start.sh`），帮你自动做好初始化。

### 上手只需要做两件事

**第一件：给自己设一个密钥。**

打开 `config` 文件夹里的 `api_keys.example.txt`，另存为 `api_keys.txt`（去掉 `.example`），然后在里面写一行你自己的密钥，格式是：

```
随便取个名字:sk-你自己编一串:备注随便写
```

比如：

```
test:sk-mykey123:我的测试密钥
```

之后在你的客户端（比如 Cherry Studio、SillyTavern 酒馆、ChatBox 等）里，把 API Key 填成 `sk-mykey123`，API 地址填成 `http://你的地址:2156/v1` 就能用了。

**第二件：记一下管理员密码。**

第一次启动时，程序会自动帮你生成一个管理员密码，**会在屏幕上打印出来**，长这样：

```
============================================================
[*] 首次启动，已自动生成管理员密码：
    密码: x3Kp9_aQ2mZv
    访问: http://<host>:<port>/admin
============================================================
```

把这个密码记下来。用它可以登录管理面板（浏览器打开 `http://你的地址:2156/admin/`），在面板里可以管理密钥、改模型、配代理，都是可视化界面，不用改文件。

> 忘了密码也没关系——打开 `config/config.json`，里面的 `admin_password` 就是。

## 在电脑上跑（Windows）

1. 把压缩包解压到你想放的地方，比如 `D:\vertex-proxy\`。

2. 双击 **`启动.bat`**。
   - 它会自动帮你初始化配置文件（如果还没有的话）。
   - 会弹出一个黑色窗口，显示运行日志。**别关它**——第一次启动时管理员密码就打印在这里。

3. 打开 `config\api_keys.txt`（用记事本就行），填上你的密钥（格式见上面），保存。

4. 关掉黑色窗口，再次双击 `启动.bat` 重新启动。

5. 浏览器打开 `http://127.0.0.1:2156/admin/`，用刚才看到的密码登录，就能看到管理面板了。

**想让它开机自动运行？** 两种方式：
- 按 `Win + R`，输入 `shell:startup` 回车，打开「启动文件夹」，把 `启动.bat` 的快捷方式放进去。
- 或者用 [NSSM](https://nssm.cc/) 注册为系统服务（更专业一点）。

**想让别的设备也能连？** 在 Windows 防火墙里放行 `2156` 端口，然后别的设备用你电脑的局域网 IP（不是 `127.0.0.1`）来连。

## 在服务器上跑（Linux）

适合在 VPS、云服务器上部署。

```bash
# 解压
unzip vertex-proxy-linux-amd64.zip -d vertex-proxy
cd vertex-proxy

# 用启动脚本跑（会自动初始化配置）
chmod +x start.sh vertex-proxy
./start.sh
```

第一次启动看屏幕上的管理员密码 → 编辑 `config/api_keys.txt` 填密钥 → 按 `Ctrl+C` 停掉再重启。

### 让它常驻后台（推荐）

压缩包里有个 `vertex-proxy.service` 文件，是 systemd 的服务模板：

```bash
# 假设你把文件放在了 /opt/vertex-proxy
sudo cp vertex-proxy.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now vertex-proxy

# 看运行状态
sudo systemctl status vertex-proxy

# 看实时日志（第一次在这里找管理员密码）
journalctl -u vertex-proxy -f
```

这样设好之后，程序会开机自动启动，崩溃了也会自动重启。

> 如果你的解压路径不是 `/opt/vertex-proxy`，记得改一下 `vertex-proxy.service` 里面的 `WorkingDirectory` 和 `ExecStart`。

### 不想折腾 systemd？简单后台也行

```bash
nohup ./vertex-proxy > vproxy.log 2>&1 &

# 看日志（找密码）
tail -f vproxy.log
```

### 加 HTTPS / 域名（可选）

在前面挂一个 Nginx 反向代理：

```nginx
location / {
    proxy_pass http://127.0.0.1:2156;
    proxy_http_version 1.1;
    proxy_set_header Connection "";
    proxy_buffering off;        # 流式输出必须关掉缓冲
    proxy_read_timeout 3600s;   # 长对话和大文件别被超时掐断
}
```

## 在手机上跑（Android）

是的，你的安卓手机也能跑。需要先装一个叫 **Termux** 的终端应用。

### 第一步：装 Termux

从 [F-Droid](https://f-droid.org/packages/com.termux/) 安装 Termux。**别从 Play 商店装**（那个版本太旧了，很多东西用不了）。

### 第二步：准备文件

打开 Termux，依次输入：

```bash
# 先更新一下 Termux 的包（可能要等一小会儿）
pkg update -y
pkg install -y unzip

# 授权 Termux 访问手机存储（会弹一个权限弹窗，点允许）
termux-setup-storage
```

然后把你的压缩包放到手机的下载目录，继续在 Termux 里输入：

```bash
# 解压（文件名换成你实际的）
cp ~/storage/downloads/vertex-proxy-android-arm64.zip ~/
cd ~
unzip vertex-proxy-android-arm64.zip -d vertex-proxy
cd vertex-proxy
chmod +x vertex-proxy
```

### 第三步：启动

```bash
sh start.sh
```

> 这里用 `sh start.sh` 而不是 `./start.sh`，因为 Termux 的文件系统路径和普通 Linux 不太一样。

第一次启动看密码 → 按 `Ctrl+C` 停掉 → 编辑密钥文件：

```bash
nano config/api_keys.txt
```

在 nano 编辑器里写入你的密钥（比如 `test:sk-mykey123:手机`），按 `Ctrl+X`，再按 `Y`，再按回车保存。然后重新 `sh start.sh`。

### 让手机别杀后台

安卓系统很喜欢杀后台。做两件事能大大改善：

```bash
termux-wake-lock
```

然后去手机的「设置 → 电池 → 应用管理」（不同品牌位置不同），找到 Termux，选「不限制后台」或「忽略电池优化」。

### 同一 WiFi 下的其它设备怎么连？

在 Termux 里输入 `ifconfig` 看你手机的局域网 IP（通常是 `192.168.x.x`），然后其它设备的 API 地址填 `http://192.168.x.x:2156/v1`。

> 这个 ARM64 版本也能在树莓派和其他 ARM 设备上跑，操作方法跟 Linux 服务器一样。

## 用 Docker 跑

如果你熟悉 Docker，可以直接用项目里的 `docker-compose.yml`：

```bash
docker compose up -d

# 看日志（找首次管理员密码）
docker compose logs -f
```

配置文件在 `config/` 目录下，会自动持久化。

## 配置怎么改

配置文件是 `config/config.json`。第一次启动时会自动生成，你也可以手动编辑它。改完**不需要重启**——程序每分钟会自动检测配置变化。

一个最基本的配置长这样：

```json
{
  "port_api": 2156,
  "max_retries": 10,
  "admin_password": "",
  "proxy_url": ""
}
```

### 常用选项

| 写什么 | 默认是 | 是干嘛的 |
|--------|--------|----------|
| `port_api` | `2156` | 程序监听的端口号。想换端口就改这里 |
| `admin_password` | 自动生成 | 管理面板的登录密码。留空会自动生成一个 |
| `max_retries` | `2` | 请求失败时自动重试几次。网络不稳可以调大点 |
| `proxy_url` | 空 | 出站代理地址（见下一节） |
| `force_no_stream` | `false` | 设成 `true` 会把所有请求都变成非流式 |

> 大部分选项都可以在管理面板（`http://你的地址:2156/admin/`）的「设置」页里用界面改，不用手动编辑文件。

### 模型相关

`config/models.json` 控制对外暴露哪些模型。已经预设好了常用的 Gemini 模型，一般不需要改。

如果你的客户端只认 `gpt-4o` 这种名字，可以在管理面板的「模型」页加一个别名映射：

```
gpt-4o  →  gemini-3.5-flash
```

这样客户端发 `gpt-4o`，实际调用的就是 `gemini-3.5-flash`。

**假流式**：有些客户端要求必须流式输出。在模型名前加 `假流式-` 或 `fake-`（比如 `fake-gemini-2.5-pro`），程序会先把完整回复生成好，再模拟成一小段一小段推送，看起来就像流式一样。

## 网络不通怎么办（代理与节点）

这个程序需要能连到 Google 的服务器。如果你的网络环境连不上（比如在国内），有三种办法：

### 方法一：挂代理

如果你本机已经有代理软件在跑（比如 Clash、v2ray），在 `config.json` 里填上它的地址：

```json
{
  "proxy_url": "socks5://127.0.0.1:1080"
}
```

或者 `http://127.0.0.1:7890`，具体看你的代理软件监听的端口。也可以在管理面板「设置」页里填。

### 方法二：用订阅节点（推荐）

在管理面板的「代理与节点」页，直接填入你的代理订阅地址或节点链接（支持 vmess、vless、shadowsocks、trojan），程序会自动解析并使用。**已经内置了 sing-box，不需要你另外装代理软件。**

### 方法三：并发竞速

如果你有多个节点，可以在管理面板开启「并发池」，程序会同时用多个节点发请求，谁先回用谁的结果。速度更快，稳定性也更好。

> 在手机上自部署时，方法二和三通常最方便——不需要在手机上另外装代理 App。

## 日常维护小贴士

- **改了配置不用重启**：改完 `config.json`、`api_keys.txt` 或 `models.json`，程序会在一分钟内自动读取新配置。

- **更新版本**：停掉程序 → 用新版的可执行文件替换旧的（`config/` 文件夹不要动，你的配置和密钥都在里面）→ 重新启动。

- **程序关掉不会丢数据**：在途的请求会等它处理完（最多 25 秒）再退出，不会突然切断。按 `Ctrl+C` 就是正常关闭。

## 遇到问题？

**所有请求都返回 401 / "missing API key"**

→ 你还没填密钥。确认 `config/api_keys.txt` 里有至少一行 `名称:sk-xxx:备注`，而且你客户端用的 API Key 就是那个 `sk-xxx`。

**管理面板登不进去**

→ 用第一次启动时打印在屏幕上的密码。忘了就打开 `config/config.json`，找 `admin_password` 那行。可以在面板「设置」页改密码。

**一直报错 / 429 / 认证失败**

→ 大概率是你的出口 IP 被 Google 限制了。参考上面「网络不通怎么办」那节挂个代理。也可以把 `max_retries` 调大一点（比如 `10`）。

**别的设备连不上**

→ 检查两件事：① 防火墙有没有放行 `2156` 端口；② 别的设备用的是你这台机器的局域网 IP（不是 `127.0.0.1`——那个只能本机访问）。

**想换端口**

→ 改 `config/config.json` 里的 `port_api`，重启程序就行。
